# claude-tmux

Launches Claude Code in a dedicated Alacritty/tmux window for a given Linear issue, triggered by opening a `.command` file.

## Flow

```
Linear automation
      │
      ▼
  .command file  ──(open)──▶  AlacrittyLauncher.app
                                      │
                                      ▼
                             claude-linear-bridge
                                      │
                          ┌───────────┴────────────┐
                          ▼                        ▼
                  tmux window                 Alacritty
                  cc:<ISSUE-ID>                (focused)
                  claude -w '<ISSUE-ID>' ...
```

The `.command` file is generated by a Linear automation and contains a `claude` invocation. Instead of running in Terminal, it gets routed into a named tmux window inside Alacritty, with worktree mode (`-w`) injected automatically.

## Prerequisites

- [tmux](https://github.com/tmux/tmux)
- [Alacritty](https://alacritty.org/)
- [Claude Code CLI](https://claude.ai/code) with `-w` (worktree) support

## Installation

### 1. Place the bridge script

The script lives at `~/.config/claude-tmux/bin/claude-linear-bridge`. Make sure it's executable:

```sh
chmod +x ~/.config/claude-tmux/bin/claude-linear-bridge
```

### 2. Create AlacrittyLauncher.app

This macOS app receives `.command` files and hands them off to the bridge:

```sh
osacompile -o /Applications/AlacrittyLauncher.app - << 'EOF'
on open theFiles
    repeat with theFile in theFiles
        set filePath to POSIX path of theFile
        do shell script "chmod +x " & quoted form of filePath
        do shell script "/Users/YOUR_USERNAME/.config/claude-tmux/bin/claude-linear-bridge " & quoted form of filePath & " >> /tmp/bridge-debug.log 2>&1"
    end repeat
    tell application "Alacritty" to activate
end open
EOF
```

Replace `YOUR_USERNAME` with your macOS username.

### 3. Fix the Info.plist (prevents Terminal.app from hijacking .command files)

Without this, macOS may revert `.command` files back to opening in Terminal after the first use:

```sh
/usr/libexec/PlistBuddy -c "Add :CFBundleIdentifier string com.local.alacritty-launcher" \
    /Applications/AlacrittyLauncher.app/Contents/Info.plist

touch /Applications/AlacrittyLauncher.app

/System/Library/Frameworks/CoreServices.framework/Frameworks/LaunchServices.framework/Support/lsregister \
    -f /Applications/AlacrittyLauncher.app
```

### 4. Set AlacrittyLauncher.app as the default for .command files

Right-click any `.command` file → **Get Info** → **Open With** → select `AlacrittyLauncher` → **Change All**.

## Configuration

| Variable              | Default   | Description                     |
|-----------------------|-----------|---------------------------------|
| `CLAUDE_TMUX_SESSION` | `claude`  | Name of the tmux session to use |

## Debugging

Logs are written to `/tmp/bridge-debug.log`.
